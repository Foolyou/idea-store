## 1. 产品概述

### 1.1 产品定义

灵感宝盒是一个灵感记录与轻社交内容平台。用户以文字与图片为载体即时捕捉灵感，并通过公开、圈子、私有三种可见范围控制内容的传播边界。

### 1.2 产品愿景

构建一个安静、高信噪比的创意孵化空间——降低发布门槛以鼓励记录，简化互动机制以抑制噪音，通过灵活的可见性控制让用户敢于表达、乐于分享。

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| 极简启动 | 昵称 + 密码即可完成注册，零验证门槛 |
| 隐私自主 | 每条内容独立控制可见范围；公开内容支持发布后即时撤回 |
| 安静互动 | 仅保留点赞与收藏，不引入评论系统，避免社交压力 |
| 移动优先 | 以移动端为第一目标尺寸，保证灵感记录的即时性 |
| 智能默认 | 发布配置继承上一次选择，降低重复操作成本 |

## 2. 目标用户

### 2.1 用户画像

- **创意工作者**：设计师、产品经理、独立开发者，需要随手记录碎片化灵感
- **兴趣社群参与者**：希望围绕特定主题与同好交流灵感的用户
- **私人笔记型用户**：将灵感宝盒作为灵感日记，仅个人可见

### 2.2 核心场景

| 场景 | 描述 | 频率 |
|------|------|------|
| 即时记录 | 突发灵感时，打开首页快速输入文字并发布 | 高 |
| 圈子浏览 | 在自己感兴趣的圈子内查看他人灵感 | 中 |
| 内容管理 | 回顾、编辑、删除自己的历史灵感 | 低 |
| 圈子发现 | 通过社区流中的圈子标签或圈子列表发现新圈子 | 中 |

## 3. 功能需求

### 3.1 用户系统

#### 3.1.1 注册

- **字段**：昵称（必填，公开显示）、密码（必填，加密存储）
- **约束**：昵称唯一，长度 2-20 字符
- **无需**：邮箱验证、手机号验证、第三方账号绑定

#### 3.1.2 登录

- 凭据：昵称 + 密码
- 登录态持久化，关闭浏览器后保持

#### 3.1.3 账号管理

- 入口：浮动菜单 → 账号设置
- 能力：修改昵称、修改密码、更换头像

---

### 3.2 灵感（Inspiration）

#### 3.2.1 数据定义

| 属性 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| id | string | 自动生成 | UUID | 唯一标识 |
| content | string | 是 | 1-5000 字符 | 文本正文 |
| images | string[] | 否 | 0-9 个 | 图片 URL 数组 |
| visibility | enum | 是 | `public` / `circle` / `private` | 可见范围 |
| circle_id | string | 条件必填 | visibility=circle 时必填 | 目标圈子 |
| author_id | string | 是 | — | 发布者 |
| created_at | timestamp | 自动 | — | 创建时间 |
| updated_at | timestamp | 自动 | — | 最近编辑时间 |
| like_count | integer | 自动 | 默认 0 | 点赞数 |
| bookmark_count | integer | 自动 | 默认 0 | 收藏数 |

#### 3.2.2 发布流程

```
[首页发布区] 
    → 输入文本内容（必填）
    → 添加图片（可选，≤9 张）
    → 选择可见范围：public / circle / private
        → 若选择 circle，指定目标圈子
    → 点击发布
    → 内容即时写入并展示
```

- **默认继承**：发布范围及目标圈子默认沿用用户上一次发布时的配置，降低连续发布的操作成本
- **即时发布**：点击发送后内容即刻上链，无需审核

#### 3.2.3 一键关闭分享

- **触发条件**：灵感以 `public` 范围发布成功后
- **交互形式**：页面底部弹出 Toast 轻提示，文案「已公开发布」，右侧附带操作按钮「关闭分享」
- **行为**：点击「关闭分享」后，该灵感 `visibility` 由 `public` 变更为 `private`，内容从社区时间线移除
- **有效期**：该 Toast 出现后保持至用户操作或页面切换，无自动消失时限

#### 3.2.4 编辑与删除

- **编辑**：任意时刻可编辑，不设时间窗口限制，不标注编辑标记
- **删除**：任意时刻可删除，删除后内容不可恢复
- **权限**：仅内容作者可编辑或删除自己的灵感

#### 3.2.5 草稿

- **自动保存**：发布区输入框内容实时自动持久化，无需手动操作
- **恢复机制**：页面重新加载或下次访问时，自动回填最近一次草稿内容
- **单副本策略**：仅保留一份草稿，新输入自动覆盖旧内容，不支持多份手动存档

---

### 3.3 可见范围

| 范围 | 枚举值 | 可见对象 | 展示位置 |
|------|--------|----------|----------|
| 公开 | `public` | 所有用户 | 首页社区流、作者个人主页 |
| 圈子 | `circle` | 指定圈子成员 | 圈子页、首页社区流（仅圈子成员可见） |
| 私人 | `private` | 仅作者本人 | 「我的灵感」列表 |

---

### 3.4 圈子（Circle）

#### 3.4.1 数据定义

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 自动生成 | 唯一标识 |
| name | string | 是 | 圈子名称，唯一 |
| description | string | 是 | 圈子简介 |
| creator_id | string | 是 | 创建者 |
| member_count | integer | 自动 | 当前成员数 |
| created_at | timestamp | 自动 | 创建时间 |

#### 3.4.2 圈子治理规则

- **创建权限**：所有已登录用户均可创建圈子
- **加入方式**：公开加入，无需审核或邀请
- **成员权限**：成员在圈内发布灵感不受限（无需管理员审批内容）

#### 3.4.3 圈子页功能

- **灵感流**：展示该圈子内所有灵感，按时间倒序
- **基本信息**：展示圈子名称、简介、成员数
- **成员列表**：展示圈子成员（头像 + 昵称）
- **发布组件**：内嵌于页面顶部，可见范围锁定为当前圈子，不可切换

#### 3.4.4 圈子发现

| 入口 | 路径 |
|------|------|
| 灵感溯源 | 社区流 / 详情页 → 点击圈子标签 → 进入圈子页 → 点击「加入」 |
| 圈子列表 | 浮动菜单 → 圈子列表 → 浏览/搜索 → 点击「加入」 |

---

### 3.5 互动系统

#### 3.5.1 点赞

- 对公开及圈子内灵感可点赞
- 无重复点赞限制（同一用户同一灵感仅计一次）
- 作者可点赞自己的灵感

#### 3.5.2 收藏

- 对公开及圈子内灵感可收藏
- 收藏内容归入「收藏夹」
- 取消收藏后内容从收藏夹移除

#### 3.5.3 互动反馈

- 不设实时推送通知
- 在个人管理页以汇总数字展示「已获点赞」与「已获收藏」的总计

#### 3.5.4 本期不包含

- 评论与回复
- 外部分享（链接 / 二维码）
- @提及与话题标签

## 4. 信息架构

### 4.1 路由规划

| 路由 | 页面 | 说明 |
|------|------|------|
| `/` | 首页 | 发布入口 + 社区时间线 |
| `/idea/[id]` | 灵感详情 | 完整内容展示 + 互动操作 |
| `/circle/[id]` | 圈子主页 | 圈子内灵感流 + 成员列表 + 发布组件 |
| `/circles` | 圈子发现 | 圈子列表 + 搜索 |
| `/me` | 个人中心 | 灵感管理 / 圈子管理 / 收藏 / 草稿 / 设置 |

### 4.2 首页布局

```
┌─────────────────────────────┐
│        发布区 (上半屏)        │
│   ┌─────────────────────┐   │
│   │  textarea (文本输入)  │   │
│   ├─────────────────────┤   │
│   │  [图片]  [范围选择]   │   │
│   │            [发布]    │   │
│   └─────────────────────┘   │
├─────────────────────────────┤
│      社区时间线 (下半屏)      │
│   ┌─────────────────────┐   │
│   │  灵感卡片 (公开)      │   │
│   ├─────────────────────┤   │
│   │  灵感卡片 (圈子)      │   │
│   ├─────────────────────┤   │
│   │  灵感卡片 (公开)      │   │
│   ├─────────────────────┤   │
│   │         ...          │   │
│   └─────────────────────┘   │
│                         ┌──┐│
│                         │⊕ ││ ← 浮动菜单球
│                         └──┘│
└─────────────────────────────┘
```

- **时间线排序**：按发布时间倒序，公开内容与圈子内容穿插展示
- **圈子内容过滤**：用户仅能看到自己已加入圈子的内容

### 4.3 浮动菜单球

- **位置**：首页右下角，`position: fixed`，不随页面滚动
- **默认状态**：收起（仅显示圆形图标按钮）
- **展开行为**：点击后弹出菜单面板

菜单项映射：

| 菜单项 | 目标 |
|--------|------|
| 我的灵感 | `/me`（默认锚定「我的灵感」标签） |
| 我的圈子 | `/me`（默认锚定「我的圈子」标签） |
| 收藏夹 | `/me`（默认锚定「收藏夹」标签） |
| 草稿箱 | `/me`（默认锚定「草稿箱」标签） |
| 发现圈子 | `/circles` |
| 账号设置 | `/me`（默认锚定「账号设置」标签） |

## 5. 数据模型总览

```
User (用户)
  ├── id
  ├── nickname
  ├── password_hash
  ├── avatar_url
  ├── last_visibility       ← 记录上次发布范围（用于默认继承）
  ├── last_circle_id        ← 记录上次目标圈子（用于默认继承）
  └── created_at

Inspiration (灵感)
  ├── id
  ├── content
  ├── images
  ├── visibility
  ├── circle_id             ← FK → Circle
  ├── author_id             ← FK → User
  ├── like_count
  ├── bookmark_count
  ├── created_at
  └── updated_at

Circle (圈子)
  ├── id
  ├── name
  ├── description
  ├── creator_id            ← FK → User
  ├── member_count
  └── created_at

CircleMember (圈子成员)
  ├── circle_id             ← FK → Circle
  ├── user_id               ← FK → User
  └── joined_at

Like (点赞)
  ├── user_id               ← FK → User
  └── inspiration_id        ← FK → Inspiration

Bookmark (收藏)
  ├── user_id               ← FK → User
  └── inspiration_id        ← FK → Inspiration
```

## 6. 非功能性需求

### 6.1 平台适配

- **第一优先级**：移动端（屏幕宽度 375px-428px）
- **第二优先级**：桌面端基础兼容（屏幕宽度 ≥ 1024px），居中最大宽度约束，不要求像素级优化

### 6.2 性能指标

| 指标 | 目标值 |
|------|--------|
| 首页首屏加载（LCP） | < 2.5s |
| 发布响应时间 | < 500ms（客户端感知） |
| 时间线滚动加载 | 每页 20 条，无限滚动 |

### 6.3 安全性

- 密码使用 bcrypt 或等效算法加密存储
- API 路由实施身份认证中间件
- 用户输入做 XSS 过滤
- 图片上传限制格式（JPEG / PNG / WebP）及文件大小（单张 ≤ 10MB）

## 7. MVP 功能边界

### 7.1 本期交付

| 模块 | 功能点 |
|------|--------|
| 用户 | 注册（昵称+密码）、登录、登录态保持、修改昵称/密码/头像 |
| 灵感 | 文字+图片发布、三种可见范围、默认配置继承、编辑、删除、自动草稿 |
| 圈子 | 创建圈子、加入圈子、圈子主页、圈子列表、搜索圈子 |
| 互动 | 点赞、收藏、已获赞/收藏数字统计 |
| 首页 | 发布区 + 社区时间线（公开+圈子穿插、时间倒序、无限滚动） |
| 详情页 | 完整内容展示 + 发布者信息 + 圈子标签 + 点赞/收藏入口 |
| 菜单球 | 浮动入口，展开/收起，六项菜单导航 |
| 个人中心 | 灵感管理、圈子管理、收藏夹、草稿箱、账号设置 |
| Toast | 公开发布后一键关闭分享 |

### 7.2 本期不交付

- 评论系统
- 外部分享（链接生成 / 二维码）
- 内容标签系统
- 审核制或邀请制入圈
- 手机号 / 邮箱 / 第三方登录
- 代码片段或链接内容格式
- 实时推送通知（WebSocket / Push Notification）
- 编辑时效限制及编辑标记
- 多份草稿手动管理
- 桌面端深度适配

## 8. 术语表

| 术语 | 英文 | 定义 |
|------|------|------|
| 灵感 | Inspiration | 平台核心内容单元，由文字和图片组成 |
| 圈子 | Circle | 围绕特定主题的兴趣社群，成员内共享灵感 |
| 可见范围 | Visibility | 灵感对外的访问控制级别：公开、圈子、私人 |
| 社区流 | Feed | 首页公开与圈子灵感的混合时间线 |
| 菜单球 | Floating Menu Ball | 首页右下角浮动交互入口 |
| Toast | Toast | 轻量级操作反馈提示，非模态 |
| 草稿 | Draft | 发布区自动保存的未发布内容 |
